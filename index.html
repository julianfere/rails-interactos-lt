Interactors

Que es un interactor?

Es una clase que se encarga de realizar una tarea especifica


  • Se usan para encapsular la logica de negocio de nuestra aplicacion
  • Cada interactor representa una accion que nuestra applicacion puede hacer

Context

Cuando un interactor es ejecutado, se le pasa un contexto el cual tiene informacion necesaria para realizar la tarea
Cuando el interactor cumple su unico proposito afecta el contexto que se le proporciono
Que significa esto? basicamente el Interactor su input es un contexto el cual es mutado y despues se retorna 
              
                context.cake = Cake.new
                context.cake.is_a_lie? # => true
Intro al context fail: Tambien podemos marcar el contexto como fallido si hay algun error durante la ejecucion del interactor
Tambien tiene definidos los siguientes metodos:
  • context.fail!
  • context.failure?
  • context.success?
                
                  context.error = "the cake was a lie"
                  context.fail! if context.cake.is_a_lie?
                
              
                
                  context.fail!(error: "the cake was a lie") if context.cake.is_a_lie?
                
              
              
                context.success? # => true
                context.fail! if context.cake.is_a_lie?
                context.success? # => false
                context.failure? # => true
